# 초기 공급(매물) 시딩 전략 — 사용자 0일 때의 대비

> 작성: 2026-06-23 · 성격: **콜드스타트 실행 전략** (공급측 시딩)
> 상위 맥락: [`chotot-competitive-analysis.md`](chotot-competitive-analysis.md) §5·§6 — "콜드스타트가 최대 적, 밀도 우선, 1개 구 파일럿"
> 대상 제품: SGR-287 HCMC 오토바이 라이더 거래 플랫폼 ([`260616_marketplace_pivot_planning_task.md`](../task/active/260616_marketplace_pivot_planning_task.md))
> 이 문서가 답하는 질문: **"권고는 알았다. 그런데 런칭일에 매물이 0이면 첫 사용자가 빈 화면을 본다. 무엇을, 누구로, 어떤 순서로 채우나?"**

---

## 0. TL;DR

- **닭-달걀에서 먼저 푸는 건 '공급(매물)'이다.** 마켓플레이스는 사는 사람보다 파는 물건이 먼저 보여야 한다. 빈 피드는 D0에 이탈을 만든다. 수요(구매자)는 **수동 포착**(찜·키워드알림)으로 충분하다.
- **우리의 진짜 무기는 이미 DB에 있다 — `repair_shop`(정비소·용품점).** 범용 마켓이 갖지 못한 "오프라인 공급원이 이미 지도에 박혀 있는" 상태다. 이들을 **앵커 셀러(anchor seller)**로 영입하는 것이 1번 전략. 신규 사용자 모집보다 빠르고 진하다.
- **가짜 유령 매물은 절대 금지.** 현재 dev seed(마이그 085/087, loremflickr 이미지)는 **실런칭 전 반드시 제거**한다. 연락하면 없는 매물은 신뢰를 한 번에 죽인다(당근도 금지). 대안은 "운영자가 실제로 보유·중개하는 진짜 재고" 또는 "검증된 정비소가 올린 실매물"뿐이다.
- **한 동네에 몰빵.** HCMC 전역이 아니라 **Bình Thạnh 1개 구**에서 임계 밀도를 먼저 만든다. 50개 분산 매물보다 1개 구 200개 매물이 거래를 만든다.
- **단일 플레이어 가치(혼자서도 쓸모)로 사용자를 먼저 데려온다.** 정비소 찾기·주유·침수 정보는 매물 0건이어도 동작한다(이미 구현됨). "도구로 들어와 네트워크로 남는다."

## 1. 닭-달걀: 어느 쪽을 먼저?

| 선택지 | 판단 |
|---|---|
| 구매자 먼저 모은다 | ❌ 살 물건이 없으면 구매자는 즉시 떠나고 다시 안 온다. 모객 비용 낭비. |
| **공급(매물) 먼저 채운다** | ✅ 물건이 보이면 구매자가 머문다. 공급은 **소수의 앵커**로 집중 제어 가능(구매자 수천 명 모객보다 통제 쉬움). |
| 양쪽 동시 | ❌ 자원 분산. 사용자 0 단계에선 한 점에 화력 집중. |

> **원칙**: 공급은 우리가 **직접 만든다(manufacture)**. 수요는 **포착(capture)**한다. 만들 수 있는 쪽(공급)에 노동을 넣고, 못 만드는 쪽(수요)은 신호만 모은다.

## 2. 공급 시딩 4대 전술 (우선순위 순)

### 전술 1 — 앵커 셀러: 정비소·용품점 영입 ⭐ (우리만의 불공정 우위)

- **왜 1번인가**: 이미 `repair_shop` 테이블에 HCMC 정비소/용품점이 위치까지 들어 있다. 이들은 *항상* 부품·용품·보호구 재고를 가진 실공급원이다. 라이더 개인 셀러를 한 명씩 모으는 것보다, **점포 1곳 = 매물 20~50건**이 즉시 들어온다.
- **실행**:
  1. 파일럿 구(Bình Thạnh)의 정비소·용품점 **10~30곳**을 오프라인 영업(필드워크). 베트남은 Zalo 기반 관계영업이 효과적.
  2. 점포에 **"검증 셀러(verified shop)" 배지** 부여 → 라이더가 신뢰하는 첫 매물 풀.
  3. 등록 부담 제거: 운영팀이 **대신 등록(concierge onboarding)** — 점포는 사진·가격만 주고, 우리가 리스팅. Chợ Tốt의 'Scan to Sell'에 대응하는 우리식 핸드온보딩.
- **신뢰 정합**: 매너온도/단골 시스템과 자연 연결. 검증 점포는 초기 매너온도 시드값(예: 40도)으로 시작 가능.
- **주의**: 이건 고가 중고 오토바이가 아니라 **저가·고빈도 부품/용품**이어야 한다(분석서 §6-4 — 고가 차량은 Chợ Tốt Xe 영역, 회피).

### 전술 2 — 운영자 큐레이션 재고 (Concierge / 정직한 단일플레이어)

- **무엇**: 운영팀이 직접 **실제 보유하거나 위탁받은 물건**을 올린다. 인기 부품(브레이크 패드, 미러, 그립, 헬멧, 우의 등) 소량을 직접 매입해 시작 재고로 사용 → 거래가 실제로 성사되게 한다.
- **❌ 절대 금지**: 보유하지 않은 가짜 매물(ghost listing). 연락 오면 없는 물건. → 신뢰 즉사. 당근도 운영정책상 금지.
- **✅ 허용**: 실제 이행 가능한(fulfillable) 매물만. "운영 공식 셀러" 계정으로 투명하게 표기.
- **출구**: 유기적 P2P 매물이 임계치를 넘으면 운영 재고 비중을 단계적으로 축소.

### 전술 3 — 기존 분산 공급의 이주 (FB 그룹 / Zalo / Chợ Tốt)

- **현실**: 라이더는 *이미* Facebook 오토바이 그룹·Zalo 단톡·Chợ Tốt에서 부품을 거래한다(분석서 §5-4). 공급은 존재한다 — 분산돼 있을 뿐.
- **실행**:
  1. 타깃 구의 활성 FB 그룹·Zalo 커뮤니티에서 **실판매자를 직접 섭외** → 우리 앱에 셀러로 온보딩(대신 등록 지원).
  2. **금지**: 타인 매물 무단 스크래핑·재게시(저작권·신뢰 문제). 반드시 **판매자 동의 하 이전**.
  3. 미끼: "FB 그룹은 검색·찜·가격알림이 없다. 여기선 네 근처 라이더가 키워드 걸어두면 자동 알림이 간다" — 셀러에게 노출 이점 제시.

### 전술 4 — 수요 신호로 공급 유도 (이미 구현된 기능 활용)

- 매물이 적어도 **수요는 미리 포착**한다. 이미 구현돼 있음:
  - **찜(listing_likes)** · **키워드 알림(keyword_alerts, 마이그 091)** · **위시리스트/검색**
- 사용자가 "○○ 헬멧 키워드 알림"을 걸면 → **수요 히트맵**이 된다. 이 데이터로 운영팀이 **무엇을 매입/영입할지** 정한다(전술 1·2 입력값).
- 즉, 빈 결과 화면을 "원하는 물건을 등록 요청/키워드 등록"으로 전환 → 이탈 대신 신호 수집.

## 3. 가짜 seed 정리 — 실런칭 전 필수 (강한 권고)

- 현재 dev에 seed된 매물(마이그 **085** 매물3 / **087** 다구 seed6, 이미지 `contents/system/marketplace/*.jpg` = **loremflickr 무작위**)은 **데모용**이다.
- **실런칭 시 그대로 두면 안 된다**:
  - 무작위 이미지 = 실물과 불일치 → 첫인상에서 "가짜 앱" 낙인.
  - 연락해도 거래 불가 → 유령도시 + 신뢰 붕괴.
- **조치**: 실런칭 데이터에서 placeholder seed 제거하거나, **전술 1·2의 실매물로 교체**. 데모 환경(`__DEV`)에만 잔존.

## 4. 지리적 집중 — 1개 구 몰빵

- 분석서 §6-2 권고 계승: **Bình Thạnh 등 1개 구** 선정.
- 이유: 거래는 **거리·밀도**의 함수다. "gần bạn(네 근처)" 피드가 첫인상인데, 매물이 전역에 흩어지면 근처에 아무것도 없다. **밀도가 거래를 만든다.**
- 임계 밀도 목표(가설, 파일럿에서 검증):
  - 활성 매물 **150~250건** / 구
  - 검증 셀러(점포) **10~30곳**
  - 주간 거래완료(약속→완료) **첫 양수 전환**
- 임계 도달 후에만 인접 구로 확장(2구 → 4구 …). 동시 확장 금지.

## 5. 셀러 인센티브 — 기존 경제 자산 재활용 (신규 구축 금지)

이미 있는 것만 쓴다:

| 자산 | 콜드스타트 활용 |
|---|---|
| **RP / 매너온도** | 초기 등록·거래완료에 RP 부여, 매너온도 시드. 단 RP 예산 캡(월 100만원) 안에서 (경제설계서 준수). |
| **BADGE_FOUNDER** (코스메틱, 이미 존재) | 파일럿 구 **초기 N명 셀러에게 "창립 셀러" 배지** → 소속감·정체성(분석서 §4 해자). |
| **단골(UserFollow 재사용)** | 검증 점포를 단골 등록 유도 → 재방문 락인. |
| **키워드 알림** | 셀러에게 "수요 있는 키워드" 노출 → 무엇을 올릴지 가이드. |

> ⚠️ 새 인센티브 시스템을 만들지 말 것. 위 자산 조합으로 충분. (CLAUDE.md Simplicity First)

## 6. 단계별 실행 계획 (검증 게이트 포함)

```
Phase 0 — 공급 제조 (런칭 D-4주 ~ D-1주)  · 목표: 빈 화면 박멸
  1. 파일럿 구 확정(Bình Thạnh) → 검증: repair_shop DB에서 해당 구 점포 수 ≥ 20
  2. 점포 10~30곳 오프라인 영입 + 대신 등록 → 검증: 활성 매물 ≥ 150건, 검증셀러 ≥ 10
  3. 운영 큐레이션 재고 확보(인기 부품 실물) → 검증: 이행 가능한 실매물만, 가짜 0건
  4. dev placeholder seed 제거 확인 → 검증: 실런칭 DB에 loremflickr 매물 0건

Phase 1 — 런칭 (D0)  · 목표: 도구로 유입, 매물로 체류
  5. 단일플레이어 가치(정비소/주유/침수)로 라이더 유입
  6. 수요 포착 ON: 찜·키워드알림·검색 → 검증: 키워드알림 등록 ≥ X건 (수요 히트맵 생성)
  7. 창립 셀러 배지 부여 → 검증: BADGE_FOUNDER 지급 = 영입 셀러 수

Phase 2 — 유동성 플라이휠 (D0 ~ D30)  · 목표: 첫 유기적 거래
  8. 수요 히트맵 → 전술 1·2로 공급 보강(데이터 주도 매입/영입)
  9. 유기적 P2P 매물 비중↑ → 운영 재고 비중↓
  10. 게이트: 주간 거래완료 양수 전환 + D7 리텐션 측정
      → 안 서면 분석서 §6-5 대로 기획 전제 재검토(라이더 버티컬 우위)
```

## 7. KPI (밀도·거래가 차별화보다 먼저)

- **공급**: 구당 활성 매물 수, 검증 셀러 수, 신규 매물/주.
- **유동성**: 주간 거래완료(약속→완료 전환), 매물당 찜·문의 수, 첫 거래까지 시간(time-to-first-trade).
- **체류**: D1/D7 리텐션, 키워드알림 등록률.
- **신뢰**: 평균 매너온도, 단골 등록 수.

> 차별화(라이더 정체성·동네 신뢰)는 **밀도가 선 다음** 측정. 밀도 없는 차별화는 무의미(분석서 §6-2).

## 8. 안티패턴 (하지 말 것)

1. ❌ **가짜/유령 매물** — 신뢰 즉사. 어떤 이유로도 금지.
2. ❌ **전역 동시 런칭** — 밀도 희석. 1개 구부터.
3. ❌ **구매자 모객 선행** — 빈 매물에 데려오면 영구 이탈.
4. ❌ **고가 중고 오토바이 정면 진입** — Chợ Tốt Xe 강점 영역(에스크로·점검·보증). 저가 부품/용품에 집중.
5. ❌ **거래 기능 경쟁** — 찜·알림·채팅으로 이기려 하지 말 것. 자본·스케일에서 진다. 해자는 관계.
6. ❌ **신규 인센티브 시스템 구축** — 기존 RP/매너온도/배지로 충분.

## 9. 리스크 (정직하게)

- **운영 노동 집약적**: 전술 1·2는 사람이 발로 뛰는 일(필드 영업·대신 등록). 자동화 불가. 초기엔 이게 정상 — Airbnb·DoorDash도 손으로 공급을 깔았다.
- **운영 재고의 자본·재고 리스크**: 매입한 부품이 안 팔릴 수 있음. 소량·인기품만, 수요 히트맵 기반으로.
- **앵커 셀러 의존**: 점포가 빠지면 공급 급감. 유기적 P2P 매물로 분산되기 전까진 취약. → Phase 2에서 P2P 전환율 추적.
- **밀도 미달 시**: 게이트(§6-10)에서 양수 거래가 안 서면 전제 재검토. "이미 있으니 하지 말자"가 아니라 "좁은 틈이 살 만큼 큰지"를 데이터로 판정(분석서 §6-5).
