## 1. 배경 / 문제의식 — §0 근거

**현 퀘스트 모델의 구조적 한계**
- 공급=비용: 미션·콘텐츠를 운영이 계속 생산 → 고갈 시 이탈.
- 보상=순수비용: RP·리워드가 수익과 무연결(적자), 유입동기 '보상 사냥' 편중 → LTV 낮음.
- 리텐션이 외부 주입 미션에 종속 → 자생 락인 약함.

**오토바이 거래 집약의 구조적 이점**
- 공급을 라이더가 생성(부품·용품·중고 오토바이 매물) → 콘텐츠 비용 0 수렴, 거래 자체가 재방문 동기.
- 위치·관계가 자연 락인(동네·단골·평판). REF-05: 후기 교환 유저 리텐션↑ = 신뢰가 락인.
- 전환비용 낮음: GPS·이미지 파이프라인·DM 직결(§6).

**HCMC 빈틈 (왜 우리)**
- 라이더의 부품·용품·정비 수요가 Facebook 그룹·정비소 직접방문으로 분산 → 신뢰·탐색·위치 매칭이 약함.
- '오토바이'라는 강한 공통관심사 + 위치 = 당근이 '동네'로 만든 신뢰·재방문을 **더 좁고 진한 버티컬**로 재현 가능. §0 결론: **성공 본질은 기술이 아니라 신뢰**.

## 2. 전환 가설 (Goal) — §0 근거

## 4. 핵심 기능 범위 (MVP) — §0 근거
> 원칙: "한 동네 첫 거래가 끝까지 도는 최소 루프"만 P0.

**P0**
- 상품 등록(REF-04): 사진(≤10) → **오토바이 카테고리**별 가이드 → 가격+제안받기 → 설명.
- **카테고리(오토바이 버티컬)**: 부품(엔진·소모품·타이어·배터리·브레이크) / 용품·보호구(헬멧·자켓·글러브·우의) / 액세서리(거치대·블박·방수·튜닝) / 중고 오토바이 매물 / (P1) 정비·수리 서비스.

- 거래 채팅(REF-05): DM 재활용 + 상품카드 + 약속 + 거래완료.
- 신뢰·프로필(REF-06): 동네 배지 + 매너온도(RP) + 후기 + 신고·차단.

**P1**: 끌올/가격내림 · 찜·키워드 알림(REF-08, SGR-274 재활용) · 필터 강화 · **정비·수리 연결**(기존 SR 정비소 DB 활용).
**P2**: 안전결제(MoMo·ZaloPay·VietQR) · 배송(Ahamove/Grab) · 정비소/기술자 비즈 프로필·광고 · 라이더 커뮤니티(REF-09).
**의도적 제외**: 결제·배송·광고·커뮤니티·다국어(초기 vi). 직거래·COD 문화상 밀도·신뢰 먼저.

## 10. 핵심 플로우 (UX) — §0 근거 재작성

> 각 플로우 REF 근거. 당근 정정 반영(인증 게이트·후기 흐름·매너온도 위치).

**A. 판매 등록 (공급) — REF-04**
1. 중앙 ➕ → (사진 먼저) 최대 10장, 첫 장 대표 → imgproxy
2. 제목 → 카테고리 → **카테고리별 입력 가이드**(필수항목·금지품목; 베트남 특화=오토바이 부품 등)
3. 가격(₫)/나눔 + **가격 제안받기(네고) 토글**
4. 동네 자동 인증(GPS=phường) → 설명 → 등록
5. 피드 즉시 노출 → (선택) 끌어올리기
- 가이드로 정보불균형·금지품목 예방(신뢰). 목표=사진~등록 최소 마찰(TTFL).

**B. 발견 → 구매 → 거래완료 (수요) — REF-02·03·05**
1. 동네 피드/검색 탐색
2. 상세(REF-03 위계) 확인
3. 찜 또는 **채팅하기**
4. 채팅: 가격 제안 → 약속(시간·장소)
5. 직거래(MVP 결제 미개입) → 상태 예약중 → **판매완료**
6. 일정 대화 후 채팅창 상단 **'거래 후기 보내기'** → 만족도(좋아요/별로예요)+매너 칭찬(선택)+텍스트
7. 매너온도 반영. '별로예요'는 상대 비노출.

**C. 신뢰 형성 (락인 엔진) — REF-01·05·06**
1. 동네 GPS 인증(배지) = 거래·채팅 진입 게이트
2. 거래완료·후기 누적 → 매너온도(36.5 시작, RP 엔진 산정) 상승
3. 매너온도·신뢰 배지 = 상세/프로필에 노출(카드 아님)
4. 신뢰 높은 판매자 우대 노출 → 단골 이웃 → 재거래율↑
- 후기 교환 유저 리텐션↑(REF-05) → 후기 자연 유도(강제 아님).

**D. 잔존 미션 (보조) — §5·REF-08**
1. 마이>미션/혜택 (또는 홈 '오늘의 동네 미션' 1슬롯)
2. 동네 미션(첫 등록/첫 거래/이웃 맺기) → 완료 RP·리워드
3. 리워드를 끌올권·노출 부스트로 소비 → 거래 루프 환류
- RP 엔진은 매너온도(C) 산정과 공유.

# ── §1. 배경 / 문제의식 ──
[현재 퀘스트 중심 모델의 구조적 한계]
- 공급=비용: 미션·콘텐츠를 운영이 계속 생산해야 함. 고갈→이탈.
- 보상=순수비용: RP·리워드가 수익과 연결 안 됨(적자 구조). 유입동기 '보상 사냥' 편중→LTV 낮음.
- 리텐션이 미션 소진에 종속: 재방문이 외부 주입 미션 의존→자생적 락인 약함.
[중고거래 전환 기대효과]
- 공급을 유저가 생성→매물=유저 콘텐츠, 콘텐츠 비용 0 수렴. 거래 자체가 재방문 동기.
- 위치·관계가 자연 락인: 같은 동네 거래·단골·평판이 쌓이면 떠나기 어려움.
- 전환비용 낮음: 보유한 GPS·동네 인프라, 이미지 파이프라인, DM이 직결(§6).
[HCMC 시장의 빈틈 — 왜 지금/왜 우리]
- Chợ Tốt: 점유율 높으나 본질이 '광고 리스팅'→스팸 매물·낮은 신뢰·동네 커뮤니티 부재·직거래 안전장치 약함.
- Facebook Marketplace: 신뢰·검색·위치 정밀도 약, 사기 노출 잦음.
- → "같은 동네의 믿을 수 있는 이웃과 안전하게 직거래" 영역이 비어 있음(당근이 한국서 점유한 자리, HCMC 무주공산).

# ── §2. 전환 가설 (Goal) ──
["당근처럼"의 우선순위 — 모호함 제거]
1. 직거래 신뢰(1순위·핵심 무기): 동네 GPS 인증 + 매너점수로 "이 사람과 거래해도 되는가" 해소.
2. 동네 생활권(2순위): 좁은 반경(phường/quận)의 살아있는 피드. "내 근처에서 지금 거래되는 것".
3. 위치기반 매칭(3순위·수단): 위치는 신뢰·생활권을 구현하는 도구이지 목적이 아님.
[핵심 가설] HCMC 도시 거주자에게 "같은 동네 안에서 신뢰할 수 있는 이웃과의 안전한 직거래"를 제공하면 Chợ Tốt 대비 거래 성사율·재방문(D7)이 유의미하게 높아진다.
[검증 지표]
- North Star: 주간 거래 성사 건수(채팅→'거래완료' 전환) — 주간 성장률 유지.
- 활성: 동네 피드 D7 리텐션 / 첫 등록까지 시간(TTFL).
- 신뢰: 채팅 응답률, 노쇼·신고율, 매너점수 분포.
- 공급: 동네당 활성 매물 수(밀도).
- ※ 임계 밀도가 가장 중요한 선행 지표. 밀도 없으면 피드가 죽고 신뢰·거래도 동반 사망 → 파일럿은 "넓게"가 아니라 "한 동네를 꽉 채우는" 전략(§7).

# ── §3. 체험 디자인 3층 (다마키 관점) ──
- 직감(첫인상): 가입·온보딩보다 동네 피드를 먼저 노출. "우리 동네(Phường)에서 지금 올라온 물건"을 거리 라벨(예 1.2km)과 함께 즉시. 'gần bạn(당신 근처)' 카피·아이콘 → "여긴 내 근처 중고장터"가 3초에 박힘.
- 놀라움(재방문): 매일 바뀌는 동네 새 매물 + "방금 OOO 거래완료" 살아있는 활동 신호. 찜 가격내림·키워드 알림·동네 인기 급상승. 잔존 퀘스트의 '오늘의 동네 미션' 1개가 가벼운 변주.
- 이야기(관계): 거래 후 매너 후기·재거래 표시·'단골 이웃'. 매너점수가 프로필에 누적되어 동네 평판이 됨. 좋은 거래가 쌓일수록 신뢰가 가시화되고, 그 신뢰가 다음 거래를 빠르게 함(선순환).

# ── §4. 핵심 기능 범위 (중고거래 MVP) ──  원칙: "한 동네에서 첫 거래가 끝까지 도는 최소 루프"만 P0.
[P0 — MVP]
- 상품 등록: 사진 여러 장(imgproxy), 제목, 가격(₫)/나눔, 카테고리, 동네 자동 태그, 설명.
- 동네 설정 + 위치기반 피드: GPS로 현재 동네(phường) 인증, 반경(내 동네/+인접) 조절, 최신순·거리순.
- 검색·카테고리: 키워드 검색 + 핵심 카테고리(전자기기/오토바이·자전거/가구/생활·주방/패션/유아). ※베트남 특화: 오토바이 부품·액세서리 비중 큼.
- 거래 채팅: 기존 DM 재활용, 상품 카드 첨부, 약속 잡기(시간·장소), 가격 제안.
- 신뢰·프로필: 동네 인증 배지, 매너점수(RP 엔진 재활용), 거래 후기, 신고·차단.
- 거래완료 처리 + 후기: 채팅 내 '거래 확정'→구매자·판매자 상호 후기.
[P1 — Fast Follow] 끌어올리기(bump)/가격 내림 · 찜 알림·키워드 알림(푸시 딥링크 SGR-274 재활용) · 필터 강화(가격대·상태·거리).
[P2 — Later] 안전결제/에스크로(MoMo·ZaloPay·VietQR) · 배송 직거래(Ahamove/Grab) · 비즈 프로필·광고형 노출(수익) · 동네생활(커뮤니티 글).
[의도적 MVP 제외] 안전결제·배송, 광고·프로모션, 비즈셀러, 커뮤니티 게시판, 다국어(초기 vi 단일). 이유: 직거래·현금/COD 문화에서 결제·배송은 초기 핵심 루프가 아니며, 밀도·신뢰부터 확보해야 함.

# ── §5. 퀘스트 보조화 방향 ──
- 진입 위치: 홈=동네 피드. 퀘스트는 5탭 중 '미션/혜택' 탭으로 격하. 홈에는 '오늘의 동네 미션' 1개 슬롯만.
- RP·보상 엔진 재활용(핵심 전략): RP를 '매너점수/신뢰'로 재배치. 거래완료·좋은 후기·빠른 응답·동네 인증 시 적립→매너온도식 지표 표시. 보상 엔진=신뢰 행동(첫 거래·후기·사기 신고 기여) 리워드 정산기.
- 리워드 사용처: 끌어올리기 무료권, 노출 부스트, (추후) 안전결제 수수료 할인 등 거래 루프 안에서 소비 → 보상이 '비용'이 아니라 '거래 촉진제'.
- 잔존 퀘스트 콘텐츠: '동네 탐험/첫 거래/이웃 맺기' 온보딩·습관화 미션으로 축소 운영. 신규 유저 첫 등록·첫 거래 깔때기 보강.

# ── §6. 기존 자산 재활용 가능성 ──
- GPS·위치(native.ts, 구역/거리 인프라): 위치기반 피드·동네 범위·동네 인증에 직결(전환 핵심 자산).
- 컨텐츠 이미지 파이프라인(contents + imgproxy): 상품 사진 업로드·썸네일·리사이즈에 그대로.
- DM(dm.py + 푸시 딥링크 SGR-274): 거래 채팅으로 재활용 + 상품 카드/약속잡기 확장.
- BFF/Engine 마이크로서비스 골격: 상품·거래·매너 도메인 신규 추가의 토대.
- 퀘스트/RP 엔진: 매너점수·신뢰 리워드 엔진으로 재배치(§5).
- 푸시·딥링크(SGR-274): 채팅·찜·가격내림·끌올 알림에 재활용.
- 계정·프로필 시스템: 매너 프로필(거래내역·후기·배지)로 확장.

# ── §7. 오픈 이슈 / 결정 대기 ──
- 서비스명·브랜드: Saigon Rider 명칭은 라이더·퀘스트 잔재라 중고거래 정체성과 충돌. 권고=신규 로컬 브랜드 분리(베트남어 chợ/khu phố/gần·이웃 뉘앙스). 코드·인프라(SGR)는 유지. 네이밍은 별도 워크샵.
- 타깃 시장: ✅결정 HCMC 로컬 동일. Chợ Tốt 대비 '동네 신뢰' 포지셔닝. 파일럿은 인구밀집·청년 비중 높은 1~2개 구(예 Bình Thạnh/Thủ Đức)에 밀도 집중 후 확장.
- 수익모델: C2C 거래수수료는 직거래 문화상 초기 부적합. 단계 권고=①무료로 밀도·신뢰 확보→②끌올·노출 부스트 유료+비즈 프로필 광고→③안전결제·배송 수수료(전자지갑)+매너 리워드 마켓.
- 결제·정산: 현금/COD vs 전자지갑(MoMo·ZaloPay·VietQR) 비중 검증 필요. MVP는 결제 미개입(직거래), P2에서 안전결제 도입.
- 모더레이션·안전: 금지품목 정책, 사기·중복 매물 탐지, 신고 처리 SLA — 신뢰 포지셔닝 생명선. 별도 정책 문서 필요.
- 언어·위치정책: 초기 vi 단일. 위치 정확도(동네 인증 신뢰도) vs 프라이버시(정확 위치 비노출, 동네 단위 표기) 균형 정책 필요.

# ── §8. 마일스톤 (초안) ──
1. 기획 확정(본 문서) → 3층 체험 + MVP 범위 합의  ←현재
2. 도메인 모델 설계(상품·거래·매너) → 스키마 초안 리뷰
3. MVP 화면 설계 → 핵심 플로우 프로토타입(등록·피드·채팅·거래완료)
4. 클로즈드 파일럿(HCMC 1개 구) → 첫 거래 성사율·D7 리텐션·동네 밀도
5. 신뢰/매너 시스템 + 잔존 퀘스트 통합 → 매너점수 분포·재거래율
6. 수익 실험(끌올·부스트 유료화) → 결제 의향·ARPU
7. 구 확장 / 안전결제·배송 도입 검토 → 신규 구 밀도 재현성

# ── §9. 정보구조 & 화면 맵 (IA) ──  5탭 바텀 내비게이션
- 🏠 홈(동네 피드): 첫인상·발견의 중심. 동네 선택·검색 진입·매물 카드(사진/가격/거리/매너)·최신·거리순.
- 🔍 검색·카테고리: 탐색. 카테고리 그리드·키워드 검색·필터(가격/상태/거리).
- ➕ 글쓰기(중앙 FAB): 공급 생성. 사진→정보 입력→동네 인증→등록.
- 💬 채팅: 거래 진행. 대화 목록·상품 카드·약속잡기·거래완료.
- 👤 마이: 신뢰·관리. 내 매물·찜·매너점수·후기·미션/혜택 진입.

# ── §10. 핵심 플로우 (UX 초안) ──
[A. 판매 등록(공급)] ➕탭→사진 선택(여러 장, 첫 장=대표) imgproxy 업로드·리사이즈→제목·카테고리→가격(₫) 또는 나눔(0₫) 토글→동네 GPS 인증/확인→설명→등록→피드 즉시 노출→(선택)끌어올리기 안내. 목표: 사진~등록 N분 이내(TTFL), 사진 먼저·텍스트 최소.
[B. 발견·구매(수요→거래)] 홈 동네 피드(또는 검색/카테고리)→상세(사진·가격·판매자 매너점수·거리·동네)→찜/채팅→가격 제안·약속 잡기(시간·장소)→직거래(MVP 결제 미개입)→채팅 내 '거래완료'→구매자·판매자 상호 매너 후기→매너점수 반영.
[C. 신뢰 형성(락인 엔진)] 동네 GPS 인증(배지)→거래 누적·후기·재거래→매너점수↑·신뢰 배지→피드·검색에서 신뢰 높은 판매자 우대 노출→'단골 이웃'→재거래율↑(선순환).
[D. 잔존 미션(보조 게이미피케이션)] 마이>미션/혜택(또는 홈 오늘의 미션 1)→동네 미션(첫 등록/첫 거래/이웃 맺기)→완료 RP·리워드 적립→끌올권·노출 부스트로 소비→거래 루프로 환류.
