## 1. TL;DR

- **공급(매물)을 먼저 만든다. 수요는 포착한다.** 빈 피드 = D0 이탈. 공급은 소수 앵커로 통제 가능, 수요는 찜·키워드알림으로 신호만 모은다.
- **우리만의 불공정 우위 = `repair_shop` DB.** 단, "정비소 재고 전부"가 아니라 **브라우징·대면확인이 실제 일어나는 품목**(헬멧·보호구·폰마운트·튜닝/중고 파츠)을 가진 **용품점·튜닝샵 중심**으로 앵커를 영입한다. 소모품(패드·오일)은 옆 가게로 걸어가지 앱으로 안 산다.
- **가짜 유령 매물 절대 금지.** dev placeholder seed(085/087, loremflickr)는 실런칭 전 제거. 운영자가 실제 이행 가능한 재고만.
- **한 거점에 몰빵.** HCMC 전역이 아니라 **라이더 밀집 거점 1곳**. 당근 기준 한 지역 **~1000 WAU**에서 거래가 자생한다(우리 좁은 버티컬엔 보정 필요).
- **신뢰축 = 별점(⭐) + 점포검증 배지.** 한국식 매너온도 체온 은유에 의존하지 않는다. 결제 안전망은 의도적으로 P2 이연(COD·직거래 문화).
- **단일플레이어 가치(정비/주유/침수)로 유입 → 명시적 다리로 마켓 전환.** "도구로 들어와 네트워크로 남는다"는 배선을 설계로 박는다.

## 2. 닭-달걀: 공급을 먼저 푼다

| 선택지 | 판단 |
|---|---|
| 구매자 먼저 | ❌ 살 물건 없으면 즉시 이탈, 모객비 낭비 |
| **공급 먼저 — 직접 제조** | ✅ 물건이 보이면 머문다. 소수 앵커로 집중 제어 |
| 양쪽 동시 | ❌ 사용자 0에서 자원 분산은 패착 |

> 원칙: 만들 수 있는 쪽(공급)에 노동을 넣고, 못 만드는 쪽(수요)은 신호만 모은다.

## 3. 당근 벤치마크 — 1000 WAU와 인센티브 사다리 (1차 자료)

당근 창업자의 초기유저 1000명 회고에서 그대로 가져올 것:

- **밀도 임계 = 한 지역 WAU ~1000명.** 그 선을 넘으면 "인위적 노력 없이 거래가 원활"해지고, 사용자가 자발적으로 물건을 올리기 시작한다. → 우리 북극성은 "매물 수"가 아니라 **거점당 WAU 임계**. (당근=한국 범용 중고. 우리는 더 좁은 버티컬 → 1:1로 못 옮기므로 파일럿에서 임계값 보정.)
- **창업자·지인 물건부터 직접 등록** → 동나면 등록 보상 이벤트 → 거래 인증 보상 → 추천 보상. 이 **노가다 사다리**가 정석(전단지 스캔하던 배민과 동일 패턴).
- **단일 밀집 거점**(판교 테크노밸리=직장 클러스터)에서 1000을 만든 뒤 인접지로 확장. **"제로 베이스가 어렵지, 물품·구매자가 어느 정도 있으면 확장은 수월"**(네트워크 효과).

→ §8 인센티브·§9 단계·§10 KPI에 이 패턴을 그대로 이식한다.

## 4. 공급 시딩 4대 전술 (우선순위 순)

### 전술 1 — 앵커 셀러: 용품점·튜닝샵 중심 영입 ⭐ (불공정 우위)

- `repair_shop` DB에 HCMC 점포가 위치까지 박혀 있다. 점포 1곳 = 매물 20~50건이 즉시 들어온다.
- **카탈로그 범위(검토 2.1 반영)**: 정비소 소모품(패드·그립·미러·오일)은 **제외** — 옆 가게로 걸어가지 앱으로 안 산다. **브라우징·비교·대면확인이 실제 일어나는 품목**(헬멧·보호구·폰마운트·액션캠 거치대·튜닝/중고 파츠)만. 앵커도 **용품점·튜닝샵**에 무게중심, 순수 소모품 정비소는 후순위.
- 실행: 거점 점포 10~30곳을 Zalo 기반 오프라인 영업 → **대신 등록(concierge)** → "검증 셀러" 배지(점포가 지도에 실재 + 위치 검증) = 1차 신뢰축.

### 전술 2 — 운영자 큐레이션 재고 (정직한 단일플레이어)

- 운영팀이 **실제 보유·위탁받은** 인기 품목을 직접 올려 거래가 성사되게 한다(당근 "창업자·지인 물건" 패턴).
- ❌ 보유하지 않은 가짜 매물 — 신뢰 즉사. ✅ 이행 가능한 실매물만, "운영 공식 셀러"로 투명 표기.
- **재고 캡(검토 2.3 반영)**: 동시 보유 매물 **≤ 30건**, 투입 자본 상한 명시, 회전 안 되는 품목 **즉시 손절**. 운영팀이 부품 소매상으로 전락하는 함정 방지. 유기적 P2P 임계 넘으면 비중 축소.

### 전술 3 — 기존 분산 공급 이주 (FB · Zalo · Chợ Tốt) — 3순위

- 공급은 이미 존재한다(분산). 거점의 실판매자를 직접 섭외 → 대신 등록.
- ❌ 무단 스크래핑·재게시 금지(판매자 동의 하 이전만).
- **한계 인지**: 구매자 0인 신생 앱으로 셀러를 끌어오는 건 공급 안의 닭-달걀. "검색·찜·알림" 미끼는 구매자 밀도가 생기기 전엔 약하다 → 전술 1·2 뒤로.

### 전술 4 — 수요 신호로 공급 유도 (구현 완료 기능)

- 찜(listing_likes)·키워드알림(091)·위시리스트 = **수요 히트맵**. 무엇을 매입·영입할지(전술 1·2 입력값)를 데이터가 정한다.
- 빈 결과 화면 → "키워드 등록" 전환으로 이탈 대신 신호 수집.

## 5. 수요 → 마켓 전환 다리 (검토 2.2 — 초안의 빠진 배선)

단일플레이어(정비/주유/침수)와 마켓이 한 앱 안에 **따로 존재**하면 유입이 거래로 안 이어진다. "침수 확인 모드"와 "부품 쇼핑 모드"는 머릿속이 다르다. 명시적 트리거를 최소 다음만큼 박는다:

- **정비소 상세 화면 → "이 가게 매물 보기"** 버튼 (앵커 점포 ↔ 매물 직결).
- **홈 / 라이딩 결과 화면 → "근처 신규 매물 N건"** 배너.
- **빈 마켓 결과 → "키워드 알림 걸기"** CTA (수요 포착).

→ 이게 전략의 핵심 배선이다. 없으면 §1의 "도구로 들어와 네트워크로 남는다"는 희망사항에 그친다.

## 6. 신뢰축 — 별점 + 점포검증 (검토 2.4 + 코드 실태 반영)

- **마켓 UI는 이미 별점(⭐ avg_rating)으로 전환됨.** 한국식 매너온도 '체온' 은유는 베트남에 보편적이지 않다 → **신뢰 표시는 별점·후기 수로 통일**한다(초안의 "매너온도 40도 시드"는 폐기).
- **앵커(점포) 1차 신뢰축 = "검증 배지 + 점포가 지도에 실재 + 위치 검증."** 별점·후기는 거래가 쌓이며 자연 누적.
- **C2C 1차 신뢰 신호 = 실물 컨디션 사진**(등록 시 사진 ≤10장, 이미 설계됨). 베트남에 보편적.
- **결제 안전망은 콜드스타트 범위 밖** — 직거래·COD 문화 전제로 의도적 P2 이연(MoMo/ZaloPay/VietQR). 콜드스타트에서 끌어오지 않는다.
- ⚠️ 별도 확인 필요: "매너온도가 베트남에 안 먹힌다"는 별점 전환 근거 리서치 문서가 저장소에 없음. chotot 분석 §4는 반대로 '관계 신뢰'를 해자로 봄 → 두 입장 정합은 §13-③에서 결정.

## 7. 거점 & 밀도 (검토 3.2 반영)

- **단위는 행정구역이 아니라 실제 근접성.** 2025년 HCMC가 168개 ward로 통폐합 → "구(district)" 개념이 데이터와 안 맞을 수 있다. 우리 코드는 **이미 ward 단위**(`wards` 테이블 + `marketplace_listings.ward_id`, SGR-310)로 가고 있으므로 **ward 데이터셋 기준으로 거점을 표현**한다.
- 거점은 행정 ward보다 **라이더 밀집 거점**(정비소 골목·배달기사 밀집지·대학가 등 = 당근의 '판교 직장 클러스터' 대응)으로 잡는다.
- 밀도가 거래를 만든다("gần bạn" 피드가 첫인상). 거점에서 임계 도달 후에만 인접지로 확장(동시 확장 금지).

## 8. 셀러 인센티브 — 3단 사다리 (당근 패턴 + 기존 경제 자산)

신규 시스템을 만들지 않고 기존 RP·쿠폰·배지로 사다리를 구성한다:

| 단계 | 트리거 | 보상(기존 자산) | 당근 원본 |
|---|---|---|---|
| ① 등록 | 매물 N건 등록 | RP / 끌올권 | 선착순 200명 샤오미 선풍기 |
| ② 거래 | **거래인증샷** 제출 | 쿠폰(커피 등) | 거래인증샷 → 무료 커피 |
| ③ 추천 | 친구 N명 초대·가입 | 쿠폰 | 친구 3명 → 스타벅스(CAC 1367원/명) |

- **창립 셀러 배지**(BADGE_FOUNDER, 구현됨)를 거점 초기 N명에게 → 소속감·정체성(해자).
- **거래인증샷**은 보상 트리거이자 **거래완료 프록시 지표**(§10) — 자기보고 버튼보다 위조 어렵다.
- RP 예산 캡(월 100만원, 경제설계서) 안에서 운용.

## 10. KPI — 밀도·유동성이 차별화보다 먼저

- **밀도(북극성)**: **거점당 WAU**(당근 ~1000 벤치마크, 파일럿 보정) · 활성 매물 150~250 · 검증셀러 10~30.
- **유동성(완료 프록시, 검토 2.5)**: 자기보고 "완료" 대신 →
  - **리스팅 소멸률**(=팔림 추정) · **매물당 문의·찜 수** · **약속 잡힌 DM 수** · **거래인증샷 제출 수** · 재방문율.
  - 단일 게이트가 필요하면 **"리스팅 소멸률 + 매물당 문의 수"**가 자기보고보다 신뢰도 높다.
- **체류**: D1/D7 리텐션, 키워드알림 등록률, 전환 다리 클릭률.
- **신뢰**: 평균 별점, 후기 수, 검증셀러 비율.

> 차별화(라이더 정체성·동네 신뢰)는 밀도가 선 다음 측정.

## 11. 안티패턴 (하지 말 것)

1. ❌ 가짜·유령 매물 — 어떤 이유로도 금지.
2. ❌ 전역 동시 런칭 — 밀도 희석. 한 거점부터.
3. ❌ 구매자 모객 선행 — 빈 매물에 데려오면 영구 이탈.
4. ❌ **정비소 소모품 카탈로그 채우기** — 안 팔리는 매물로 빈 피드 채우는 건 안 푼 것과 같다.
5. ❌ 고가 중고 오토바이 정면 진입 — Chợ Tốt Xe 강점 영역(에스크로·점검·보증).
6. ❌ 거래 기능 경쟁 — 찜·알림·채팅으로 이기려 말 것. 해자는 관계.
7. ❌ 신규 인센티브 시스템 구축 — RP·쿠폰·배지로 충분.
8. ❌ 매너온도 체온 은유 재도입 — 별점으로 통일.

## 12. 리스크 (정직하게)

- **운영 노동 집약**: 전술 1·2는 발로 뛰는 일(필드 영업·대신 등록). 자동화 불가 — 당근·배민도 손으로 깔았다. 단 우리는 25~30% 완성 단계의 비-펀딩 팀 → §13 블로커로 규모를 좁혀야 한다.
- **앵커 PMF 미검증(최대 리스크)**: "베트남 라이더가 부품·용품을 로컬 앱으로 사는가"는 가설. §13-①에서 카탈로그 범위로 좁혀 검증.

- **밀도 미달**: 게이트(§9-11)에서 양수 거래·WAU가 안 서면 전제 재검토.
