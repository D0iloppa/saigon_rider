# 동네지도 UI/UX 정의

## 목적

동네지도는 사용자가 **내 주변 거래와 피드를 빠르게 훑고**, 필요할 때 **특정 동네를 좁게 보거나 지도 범위 전체를 넓게 탐색하는 화면**이어야 한다.

이 화면의 핵심 가치는 다음 3가지다.

1. 지금 내가 어디를 보고 있는지 즉시 이해된다.
2. 상품과 피드를 지도와 목록에서 자연스럽게 오갈 수 있다.
3. 좁게 보기와 넓게 보기를 사용자가 의도적으로 전환할 수 있다.

---

## 현재 UX에서 확인된 문제

### 1. 범위 기준이 불명확하다

- 사용자는 지금 결과가 `선택한 동 기준`인지 `현재 보이는 지도 범위 기준`인지 바로 알기 어렵다.
- 범위 전환이 토글 형태로 숨어 있으면 의미를 학습해야 한다.

### 2. 첫 진입 상태가 약하다

- 화면에 들어왔을 때 무엇을 해야 하는지 안내가 약하면 빈 화면처럼 느껴질 수 있다.
- GPS 실패 시에도 사용자가 바로 탐색을 이어갈 수 있어야 한다.

### 3. 지도와 시트의 관계가 덜 명확하다

- 지도에서 마커를 눌렀을 때 시트가 어떻게 반응하는지 일관성이 중요하다.
- 시트 헤더만 보고도 현재 동네, 현재 탭, 현재 범위를 이해할 수 있어야 한다.

### 4. 탐색 모드가 섞여 있다

- 사용자는 실제로 두 가지 행동을 한다.
- `이 동네만 자세히 보기`
- `지금 보이는 지도 범위 전체 훑기`

이 두 모드는 UX에서 명시적으로 분리되어야 한다.

---

## 제품 관점에서의 역할 정의

동네지도는 검색 화면이 아니라 **탐색 화면**이다.

- 검색 화면은 사용자가 키워드와 필터를 먼저 준다.
- 동네지도는 사용자가 위치와 지도를 먼저 본다.

따라서 동네지도는 아래 순서로 설계하는 것이 맞다.

1. 위치 맥락을 먼저 보여준다.
2. 그 위치 안의 결과를 보여준다.
3. 더 넓게 볼지 더 좁게 볼지 사용자가 전환한다.
4. 상세 탐색은 마켓 상세나 피드 상세로 넘긴다.

---

## 목표 UX 원칙

### 1. 항상 현재 맥락을 헤더에 노출한다

헤더에서 최소 3가지를 보여줘야 한다.

- 현재 보고 있는 동네 이름
- 현재 보고 있는 데이터 종류: 상품 / 피드
- 현재 범위: 선택한 동 / 지도 범위

### 2. 기본값은 좁은 범위다

첫 진입 또는 동 선택 직후 기본 모드는 `선택한 동`이어야 한다.

이유:

- 결과가 예측 가능하다.
- 데이터가 덜 흔들린다.
- 사용자가 “왜 저 멀리 있는 결과가 보이지?”라고 느끼지 않는다.

### 3. 넓게 보는 모드는 명시적 전환이어야 한다

`지도 범위` 모드는 사용자가 의도적으로 켜야 한다.

이유:

- 팬/줌만으로 결과가 계속 바뀌면 피로하다.
- 목록이 갑자기 바뀌는 느낌은 신뢰를 떨어뜨린다.

### 4. 마커 클릭은 항상 시트와 연결된다

마커 클릭 시:

- 해당 카드가 있는 탭의 시트를 연다.
- 해당 아이템으로 스크롤한다.
- 선택 상태를 시각적으로 강조한다.

### 5. 빈 상태도 행동을 제안해야 한다

빈 결과는 단순히 “없어요”로 끝나면 안 된다.

- 선택한 동 모드에서 비면: `지도 범위로 넓혀보기`
- 지도 범위 모드에서 비면: `다른 동 선택하기` 또는 `내 위치 재탐색`

## 권장 정보 구조

화면은 3개 레이어로 본다.

### 1. 지도 레이어

역할:

- 현재 위치 맥락 제공
- 지역 선택
- 결과 위치 분포 확인

포함 요소:

- 지도 본체
- 내 위치 버튼
- 줌 컨트롤
- 동 경계 강조
- 줌아웃 시 집계 배지
- 줌인 시 개별 마커

### 2. 시트 헤더 레이어

역할:

- 현재 상태를 압축해서 설명
- 주요 전환 액션 제공

포함 요소:

- 상품 / 피드 세그먼트 탭
- 현재 동네 칩
- 범위 전환 스위치
- 결과 개수

### 3. 시트 본문 레이어

역할:

- 실제 탐색 결과 소비

포함 요소:

- 상품 카드 목록 또는 피드 카드 목록
- 선택 항목 강조
- 광고 삽입
- 로딩 상태
- 빈 상태

## 추천 상호작용 모델

### A. 첫 진입

권장 흐름:

1. 저장된 마지막 동네가 있으면 우선 복원
2. 없으면 GPS 시도
3. GPS 성공 시 해당 동으로 진입
4. GPS 실패 시 기본 동네로 시작
5. 사용자가 직접 다른 동을 탭할 수 있게 유지

첫 진입에서 보여줄 문구:

- “현재 위치를 찾는 중입니다”
- “직접 동네를 탭해 시작할 수도 있어요”

### B. 동 선택

동을 탭하면 즉시 아래가 일어나야 한다.

1. 선택 동 강조
2. 범위를 `선택한 동`으로 고정
3. 목록 새로고침
4. 시트 헤더에 동 이름 반영

### C. 범위 전환

두 모드를 명시적으로 둔다.

- `선택한 동`
- `지도 범위`

동작 원칙:

- `선택한 동`: 팬/줌을 해도 결과 기준은 유지
- `지도 범위`: 현재 viewport 기준으로 결과 갱신

### D. 마커 클릭

마커 클릭 시:

1. 해당 카드 선택
2. 시트 확장
3. 해당 카드로 스크롤
4. 피드면 확장 가능 상태 유지

### E. 탭 전환

상품/피드 전환 시:

- 범위와 동네는 유지
- 선택 마커/선택 카드 상태는 초기화
- 결과 개수는 새 탭 기준으로 갱신

## 권장 컴포넌트 구성

### 지도 상단/우측 컨트롤

- 우상단: 줌인 / 줌아웃
- 우하단: 내 위치

원칙:

- 범위 전환은 지도 플로팅 버튼이 아니라 시트 헤더에 둔다.
- 지도 버튼은 “지도 조작”만 담당한다.

### 시트 헤더

권장 순서:

1. 상품 / 피드 탭
2. 결과 개수
3. 현재 동네 칩
4. 범위 전환 세그먼트

이유:

- 사용자가 가장 자주 보는 정보가 위에 있어야 한다.
- 범위 전환은 상태 정보와 같이 보여야 의미가 선명하다.

## 상태별 UX 정의

### 1. 로딩

- 지도는 유지
- 시트에는 skeleton 노출
- 기존 결과가 있으면 완전 비우기보다 자연스럽게 교체

### 2. 빈 상태

#### 선택한 동 모드에서 빈 경우

- “이 동네에는 아직 결과가 없어요”
- 보조 액션: `지도 범위로 보기`

#### 지도 범위 모드에서 빈 경우

- “현재 지도 범위에 결과가 없어요”
- 보조 액션: `내 위치로 이동`

### 3. 오류 상태

- 전체 화면 에러보다 부분 실패가 낫다.
- 지도는 유지하고 시트에 재시도 액션 제공

문구 예시:

- “결과를 불러오지 못했어요”
- “다시 시도”

## 데이터 동작 원칙

### 기본 기준

- 동 선택 직후: polygon 기준 필터
- 지도 범위 모드: viewport bbox 기준 필터

### 개수 표기

- 선택한 동 모드: 현재 동 기준 개수
- 지도 범위 모드: 현재 화면 기준 개수
- 줌아웃 상태: 동별 집계 배지 표시

## 최종 권장 UX 결론

동네지도는 아래 한 문장으로 정의할 수 있어야 한다.

> “기본은 내가 선택한 동네를 정확히 보고, 필요할 때만 현재 지도 범위로 넓혀보는 탐색 화면”

이 정의를 기준으로 하면 최선의 UX는 다음과 같다.

1. 첫 진입 시 저장 위치 또는 GPS로 자동 시작한다.
2. 동을 선택하면 기본적으로 `선택한 동` 모드로 잠근다.
3. `지도 범위`는 시트 헤더에서 명시적으로 전환한다.
4. 시트 헤더만 보고도 현재 동네, 현재 탭, 현재 범위를 이해할 수 있게 한다.
5. 마커와 목록은 항상 동기화한다.
