# 월드맵 페이지 개편 기획

## 1. 배경

현재 월드맵(/home) 페이지는 호치민 구역 SVG 지도와 퀘스트 핀, 게임 통화 카드로 구성되어 있다.
서비스 초기 단계에서 유저 유입을 위해서는 **게임 요소보다 실용적 정보 제공**이 더 효과적인 hook이 된다.

베트남 오토바이 라이더가 매일 확인하고 싶은 정보를 제공하면, 자연스러운 앱 진입 동기가 만들어진다.

## 2. 목표

| 항목 | 내용 |
|------|------|
| 핵심 가치 | **"라이더에게 유용한 정보를 매일 제공한다"** |
| 1차 지표 | 일일 홈 화면 방문율 (DAU/MAU) |
| 2차 지표 | 홈 → 퀘스트 전환율 |

## 3. 대상 사용자

- 베트남 호치민 거주 오토바이 라이더
- 출퇴근 및 배달 업무 종사자 (Grab, Shopee Food 등)
- 일상적으로 오토바이를 주 교통수단으로 사용하는 사람

## 4. 제공 정보

| 정보 | 라이더에게의 가치 | 우선순위 |
|------|-------------------|----------|
| **날씨** (기온, 비, 바람) | 외출 전 우비/장비 준비 판단 | Phase 1 |
| **대기질** (AQI) | 마스크 착용 판단, 건강 관리 | Phase 1 |
| **유가** (Ron 95/92/E5) | 주유 타이밍 & 비용 관리 | Phase 1 |
| 주유소 위치 | 가까운 주유소 찾기 | Phase 2 |
| 침수 경보 | 우기(5~11월) 침수 구간 회피 | Phase 2 |
| 교통 상황 | 출퇴근 경로 선택 | Phase 3 |

### 제거 요소

| 기존 요소 | 이유 | 대체 접근 경로 |
|-----------|------|---------------|
| SVG 구역 지도 + 퀘스트 핀 | 정보성 부족, 시각적 비중 과다 | 퀘스트 탭에서 접근 |
| 통화 카드 (XP/Gold/SP) | 게임 요소 → 정보 허브와 성격 불일치 | 프로필/Game Hub에서 접근 |
| 인사 메시지 + 레벨 프로그레스 바 | 헤더 간소화 | 프로필에서 상세 확인 |

## 6. 단계별 로드맵

```
Phase 1 ──────────────────────────────────────────
  Mock 데이터 기반 UI 완성
  · 날씨 위젯 (기온/바람/습도/비 예보)
  · 대기질(AQI) 인디케이터
  · 유가 위젯 (3종 연료가격)
  · 관리자 유가 관리 페이지

Phase 2 ──────────────────────────────────────────
  실제 외부 API 연동
  · 날씨 API 연동 (WeatherAPI.com 등)
  · Redis 캐싱 (서버 부하 방지)
  · 주유소 위치 지도
  · 우기 침수 경보

Phase 3 ──────────────────────────────────────────
  고급 정보 서비스
  · 실시간 교통 정보
  · 개인화 경로 추천
  · 커뮤니티 리포트 (침수/사고 제보)
```

## 7. 기술 의사결정

| 항목 | 결정 | 이유 |
|------|------|------|
| Phase 1 데이터 | Mock 데이터 | UI/UX 검증 우선, API 비용 발생 전에 화면 확정 |
| 날씨 API | WeatherAPI.com (후보) | 무료 1M calls/월, 시간별 예보 포함, HCMC 지원 |
| 유가 데이터 | DB + 관리자 수동 입력 | 베트남 유가 정부 고시, ~10일 간격 변경, 공식 API 없음 |
| 날씨 캐싱 | Redis (10분 TTL) | 이미 인프라에 Redis 존재, 외부 API 호출 최소화 |
